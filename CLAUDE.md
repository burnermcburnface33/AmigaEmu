# AmigaEmu ‚Äî iOS Commodore Amiga emulator (vAmiga core)

Third app in the family (after dospad/DOSBox-X and AppleIIGS/KEGS), same architecture: a C++ emulator core driven by an Obj-C++ bridge on a background pthread, wrapped in a SwiftUI/UIKit shell. Built with xcodegen. Target/scheme `AmigaEmu`, display name "Amiga", bundle `com.boris.amiga`, iOS 17+.

## Core = vAmiga
- `AmigaEmu/Core/` is a verbatim copy of `/Users/boris/Downloads/GS/Amiga/vAmigaWeb/Core` (vAmiga v4.3.1, **C++20**). Public facade: `vamiga::VAmiga` (`Core/VAmiga.h`).
- The web build's `main.cpp` + `Headless.cpp` are the call-pattern references (NOT compiled ‚Äî they have `main()`).
- **Full integration map: `INTEGRATION_PLAN.json`** (the 6-lens mapping + synthesis). Read it before touching the bridge.

## Threading (the load-bearing fact)
This Core fork's internal emulator thread is **commented out** (`Thread.cpp:24`; `isLaunched/isEmulatorThread/isUserThread` hardcode `true`). So it's **host-driven**: `EmulatorBridge` owns ONE pthread that calls `_emu->emu->computeFrame()` paced by a wall-clock budget at 50 fps (PAL), with `_emu->emu->update()` every tick. Control ops (disk/reset) are serialized onto that pthread (`enqueueControl:`); input goes through the thread-safe `_emu->put(Cmd::‚Ä¶)` queue; the renderer reads `videoPort.getTexture()` under `lockTexture/unlockTexture`.

## Build specifics
- `CLANG_CXX_LANGUAGE_STANDARD: c++20` (needs std::latch/concepts). Recursive `HEADER_SEARCH_PATHS: $(SRCROOT)/AmigaEmu/Core/**` (flat includes in .cpp + prefixed in VAmigaTypes.h).
- Excludes (project.yml): all `CMakeLists.txt` (√ó40 ‚Äî else "Multiple commands produce" resource collision) + `Headless.cpp/.h` + `ThirdParty/xdms/{xdms.c,_old_pfile.c}`. Compile everything else ‚Äî `Amiga` owns `RemoteManager`, so RemoteServers/Recorder can't be dropped (inert at runtime). NO stubs file needed.
- ROM/disk are folder references ‚Üí looked up with `inDirectory:@"ROMs"` / `subdirectory:"Disks"`.

## Framebuffer
`videoPort.getTexture(&nr,&lof,&prevlof)` ‚Üí RGBA8888, **912√ó313**, stride 3648. The visible picture is a crop; `MetalRenderer.cropRect` (UV x0,y0,x1,y1) auto-tunes via `videoPort.findInnerAreaNormalized` (bridge `-visibleCropLeft:top:right:bottom:`), then **LOCKS once it settles** (`cropLocked` after ~3 steady samples) so the image doesn't "resize" every time the Amiga changes video mode. `resetAutoCrop()` re-detects, called by the controller on reboot/clean-boot. Source texture `.rgba8Unorm`, framebuffer `.bgra8Unorm`. Display aspect 4:3.

## Machine config (Amiga 2000) ‚Äî `EmulatorBridge.start()`
vAmiga 4.3.1 is **OCS/ECS only**: `CPURev` tops out at `CPU_68EC030` (no 68040), `AgnusRevision` at `ECS_2MB`, `DeniseRev` at `ECS` ‚Äî **no AGA**, so A1200/A4000 are impossible. The machine is built as a loaded **Amiga 2000** (no A2000 ConfigScheme exists ‚Äî start from `A500_ECS_1MB`, then override): `CPU_68000`, `AgnusRevision::ECS_1MB` (8372A), `DeniseRev::OCS` (8362; ECS Denise is "only partially supported"), `BankMap::A2000B`, `TV::PAL`, and the core's **max RAM** = 1 MB chip (`MEM_CHIP_RAM 1024`, the ECS_1MB Agnus ceiling) + 1.5 MB slow (`MEM_SLOW_RAM 1536`) + 8 MB Zorro-II fast (`MEM_FAST_RAM 8192`) = **10.5 MB**. RAM/chipset opts are `OPT_LOCKED` once powered on ‚Üí set them before `powerOn()`, and `set()` is async so drain with `_inner->update()`. ROM: loads the bundled `kickstart-3.1-a500_a600_a2000.rom` (40.063 = the genuine A500/A2000 ROM) but **falls back to any `*.rom` in `Resources/ROMs`** so a user-supplied Kickstart just works.

### "zoom-in on rotate/restore" ‚Äî FIXED (2026-06-02)
The auto-crop **locks** after converging (`MetalRenderer.cropLocked`); gesture `zoomScale`/`panOffset` also persist. So on rotate the locked crop didn't re-fit the new aspect, and on restore it was stale ‚Üí looked zoomed-in. **Fixed:** (1) `EmulatorController.loadSaveState(_:)` + `resumeSavedSession()` now call `metalRenderer?.resetAutoCrop()` after `bridge.loadState‚Ä¶`. (2) `EmulatorScreenView.layoutSubviews()` tracks `wasLandscape`; on an orientation flip it resets `zoomScale=1.0`/`panOffset=.zero` + `resetAutoCrop()` so the crop re-converges for the new aspect. Rotate verified on sim (landscape re-frames correctly).

### Mechanical drive sound effects removed (2026-06-02)
The floppy click / insert / eject SFX were eliminated at the user's request. There were **two** sources, both silenced: (1) host-played samples ‚Äî `bridgeListener` no longer dispatches `Msg::DRIVE_STEP/DISK_INSERT/DISK_EJECT` to `playDrive*`, and `startAudio` no longer calls `loadDriveSounds` (the `Resources/Sounds/*.mp3` + `playDrive*`/`loadDriveSounds` remain in the tree but unused); (2) vAmiga's own internal synthesis ‚Äî `start()` now sets `Opt::DRIVE_STEP/POLL/INSERT/EJECT_VOLUME = 0` (defaults were 50/0/50/50 per `Defaults.cpp`), so the core mixes no drive sounds into `copyStereo` either. Paula game audio is unaffected; the disk-activity LED (driven by `DRIVE_MOTOR`/`DRIVE_WRITE`) still works.

### Faster disk access (2026-06-02)
A real Amiga floppy is intentionally slow, so game loads took ages. `EmulatorBridge.start()` now sets `Opt::DC_SPEED = 8` (DiskController acceleration: `1` = authentic, `2/4/8` = faster disk-DMA, `-1` = instant "turbo"). 8√ó keeps vAmiga's timing model intact (safer for copy-protected titles than turbo) while loading ~8√ó faster. Bump to `-1` if a user wants instant loads and accepts the small protection-compatibility risk. (dospad/AppleIIGS disk handling was already correct ‚Äî dospad copies imports into DriveC synchronously under scoped access, no async/scoped-URL bug, so no fix was needed there.)

### Imported disks now actually load ‚Äî FIXED (2026-06-02)
**Bug:** importing an ADF via the Files picker showed the disk as mounted in DF0 (UI) but the Amiga sat on Kickstart's "insert disk" screen ‚Äî the disk never reached vAmiga. Root cause: `EmulatorController.insertDisk(at:)` passed the **security-scoped picker URL straight to the bridge**, but `EmulatorBridge.insertDiskAtPath:` only **enqueues** the insert onto the emu thread and returns `YES` immediately (before `swapDisk`). By the time the queued block ran `FloppyFile::make(path)` on the emu thread, the Swift function had already returned and `stopAccessingSecurityScopedResource()` had fired, so the path (often an un-materialised Files temp) was unreadable ‚Üí `make()` threw ‚Üí caught/logged ‚Üí DF0 left empty. The bundled disk worked only because its bundle path is always readable. **Fix:** `insertDisk(at:)` now **copies the picked file into `Documents/Disks/` while still holding scoped access** (new `diskLibraryDirectory()`), then inserts from that stable local path ‚Äî matching the AppleIIGS/dospad import model, and persisting the disk. vAmiga's `FloppyDriveAPI::insert ‚Üí swapDisk` already handles disk-change correctly (ejects any existing disk, inserts with the modeled delay), so once the path is readable the Amiga detects and boots it. Verified on the iPad sim end-to-end: eject + hardReset to a blank Kickstart "insert disk" screen, then import an ADF from an **external temp file** through the fixed path ‚Üí the volume mounts and the Amiga reads it. (Follow-up, not done: list previously-imported disks from `Documents/Disks` in `DiskManagerView` so they're re-selectable.)
